## 三、回调验签机制（核心修复）

### 3.1 问题回顾

```
5月7日凌晨发现：所有收钱吧回调被拒
├─ RSA验签：全部 FAIL
│  └─ 根因：Node 24 / OpenSSL 3.x 无法解析收钱吧 RSA 公钥
│     （DECODER routines::unsupported）
└─ MD5降级：也失败
   └─ 根因：回调 body 无 sign 字段，
      错误地把 Authorization header 的 RSA base64 与 MD5 hash 对比
```

### 3.2 修复方案（三阶段验签）

```
┌─────────────────────────────────────────────┐
│         收钱吧回调验签流程 v2.0              │
├─────────────────────────────────────────────┤
│                                             │
│  ╔═══════════════════════════════════════╗  │
│  ║ 第一优先：MD5 验签                    ║  │
│  ║ body.data.sign 字段                  ║  │
│  ║ terminal_sn → 查DB → terminalKey    ║  │
│  ║ MD5(参数排序+key) vs data.sign      ║  │
│  ╚══════╦═══════════════════════════════╝  │
│         │ 失败                             │
│         ▼                                  │
│  ╔═══════════════════════════════════════╗  │
│  ║ 第二备选：RSA 验签                    ║  │
│  ║ Authorization header                ║  │
│  ║ crypto.createVerify('RSA-SHA256')    ║  │
│  ╚══════╦═══════════════════════════════╝  │
│         │ 失败                             │
│         ▼                                  │
│  ╔═══════════════════════════════════════╗  │
│  ║ 最后防线：IP 信任                     ║  │
│  ║ true-client-ip 匹配 47.96.x.x       ║  │
│  ║ (SQB 阿里云杭州固定IP段)             ║  │
│  ╚═══════════════════════════════════════╝  │
│                                             │
│  修复代码：src/routes/shouqianba.db.js      │
│  Git commit：1f4ab33                        │
│  Render部署：自动（GitHub law-backend）      │
└─────────────────────────────────────────────┘
```

### 3.3 验证方法

- **MD5**：WAP 支付回调 body 自带 `sign` 字段，terminalKey 从数据库 `shouqianba_terminals` 表获取
- **RSA**：Card 支付可能用 Authorization header，保留兼容
- **IP 信任**：SQB 回调 IP 固定 `47.96.x.x`（阿里云杭州），伪造风险极低

## 五、风险提示

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| RSA 公钥过期/不匹配 | 低 | 已设置 MD5 + IP信任双重兜底 |
| terminalKey 轮换 | 中 | 签到后自动更新缓存；下次付款失败时重新签到 |
| Render 冷启动回调延迟 | 低 | 前端3秒轮询 + 「我已支付」手动按钮兜底 |
| 网络波动回调丢失 | 低 | 收钱吧有重试机制；用户可点「我已支付」手动确认 |
