## 必须修复的问题（按优先级）

### 🔴 P0-1: 密码明文存储（严重安全漏洞）
**文件**: `src/routes/auth.min.js`
**问题**: 登录/注册接受 `password` 参数，但**完全没有加密**，密码明文存储
**影响**: 数据库泄露则所有用户密码泄露

**修复步骤**:
1. 在文件顶部添加：`import bcrypt from 'bcryptjs';`
2. 在注册接口中，保存前加密密码：
   ```javascript
   // 注册时加密密码
   const hashedPassword = await bcrypt.hash(password, 12);
   // 保存用户时存入 hashedPassword
   ```
3. 在登录接口中，验证密码：
   ```javascript
   // 找到用户后，验证密码
   if (user.hashedPassword) {
     const valid = await bcrypt.compare(password, user.hashedPassword);
     if (!valid) return res.status(401).json({ success: false, error: '密码错误' });
   } else if (user.password === password) {
     // 兼容旧明文密码（迁移）
     const valid = true;
   } else {
     return res.status(401).json({ success: false, error: '密码错误' });
   }
   ```
4. 确保 `data/users.json` 中新用户有 `hashedPassword` 字段

**验收**:
1. 注册新用户 → `data/users.json` 中密码字段是加密的（如 `$2a$12$...`）
2. 用正确密码登录 → 成功
3. 用错误密码登录 → 失败

### 🔴 P0-2: 认证中间件数据源不一致
**文件**: `src/middleware/auth.js`
**问题**:
- `authMiddleware` 使用 `findUserById` 从 `dataStore.js` 读取用户
- 但 `auth.min.js`（实际生产auth）将用户存储在 `data/users.json` 和 PostgreSQL
- `dataStore.js` 和 `data/users.json` 不同步，导致登录后调用认证API失败

**修复步骤**:
修改 `src/middleware/auth.js`，改用 `dbService.js` 的 `findUserById`：
```javascript
// 原来（从 dataStore.js）：
import { findUserById, findUserByEmail, updateUser } from '../services/dataStore.js';

// 改为（从 dbService.js）：
import { findUserById } from '../services/dbService.js';
```

同时需要：
1. 确保 `dbService.js` 有 `findUserById` 函数
2. 确保 PostgreSQL users 表能查到 auth.min.js 创建的用户
3. auth.min.js 在注册/登录时会 `syncUserToPostgres`（异步），需确保 PostgreSQL 写入成功

**特别注意**：
- `auth.min.js` 第117行：`'UPDATE users SET name=$1, membership_type=$2, updated_at=NOW() WHERE email=$3'`
- PostgreSQL users 表的 UNIQUE 字段可能是 `email` 而非 `id`
- 需要确认 `dbService.findUserById` 的查询逻辑

**验收**:
1. 注册并登录后，调用 `GET /api/membership`（需认证）→ 返回会员信息（不是401）
2. 登录token在7天内持续有效
