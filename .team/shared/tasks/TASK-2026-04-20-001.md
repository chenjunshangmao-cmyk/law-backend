## 深度分析结果（总指挥完成）

### 关键发现：生产入口不是 index.js！

- `package.json` 指定 `"main": "src/index.db.js"`
- `src/index.db.js` 是**实际生产入口**（而非 `src/index.js`）
- `index.db.js` 已包含 payment 路由挂载 ✅

### 🔴 P0 - 致命问题

#### 问题1：密码明文存储（严重安全漏洞）
- **文件**: `src/routes/auth.min.js`（实际生产auth）
- **现象**: 登录/注册接受 `password` 参数，但**完全没有加密存储**
- **影响**: 数据库泄露则所有用户密码泄露
- **修复**: 使用 `bcryptjs`（已在 package.json 中）加密存储：
  - 注册时：`bcrypt.hash(password, 12)`
  - 登录时：`bcrypt.compare(password, storedHash)`

#### 问题2：认证中间件数据源不一致
- **文件**: `src/middleware/auth.js`
- **现象**: `authMiddleware` 使用 `findUserById` 从 `dataStore.js` 读取用户
  - 但 `auth.min.js`（实际生产auth）将用户存储在 `data/users.json` 和 PostgreSQL
  - `dataStore.js` 可能无法找到 auth.min.js 创建的用户
- **影响**: 登录后调用需要认证的API（`/api/membership`、`/api/accounts` 等）会返回 401 用户不存在
- **修复**: 修改 `authMiddleware`，统一使用 PostgreSQL 查询用户（通过 `dbService.js`）

### 🟡 P1 - 重要问题

#### 问题3：会员升级数据不一致
- **文件**: `src/routes/membership.db.js`
- **现象**: `upgradeUserMembership` 更新 PostgreSQL，但 `authMiddleware` 读 `dataStore.js`
- **修复**: 与问题2同根源，统一数据源即可解决

#### 问题4：收钱吧生产终端
- **文件**: `src/routes/payment.db.js`
- **现象**: 使用测试终端 `91803325`
- **状态**: 开发阶段可接受，生产需真实商户配置

### 🟢 P2 - 架构问题

#### 问题5：Playwright 浏览器安装
- `package.json` 已声明 `"playwright": "^1.59.1"`
- Render 环境可能未安装 Chromium，需在 postinstall 中添加 `npx playwright install chromium`
