# 2026-04-24 工作日志

## 支付系统重大修复

### 问题：401 认证错误
所有付费端点（`/api/payment/create`）返回 401：用户不存在或会话已失效。

**根因**：ESM 模块导入绑定问题。`database.js` 中 `let pool = null` 在异步 IIFE 里赋值，但 `dbService.js` 导入时 `pool` 还是 `null`。

### 修复1：authMiddleware 直接查 pool
文件：`backend/src/middleware/auth.js`（已同步到 `src/middleware/auth.js`）
- 绕过 `dbService.findUserById`，在 authMiddleware 直接用 `pool.query` 查用户
- 降级 fallback 保留 dbService 作为 JSON 兜底

### 修复2：简化支付 + 会员系统
文件：`backend/src/routes/payment.db.js`、`backend/src/routes/membership.db.js`

**新架构（用户方案）**：
- 付款链接只负责收款 → 记录交易 → **不自动激活会员**
- 会员激活由 AI客服 调用 `/api/membership/check-and-activate` 统一处理
- 4个会员开关：`MEMBERSHIP_ENABLED = { basic: true, premium: true, enterprise: true, flagship: true }`
- 自动计算 `paidUntil`（付款时间 + 套餐天数）
