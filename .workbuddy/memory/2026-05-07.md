## 收钱吧回调验签修复 (v2026.05.07.008) ⚡

### 问题
用户付款后回调数据正确到达（支付宝 PAID），但 RSA 验签全部 FAIL，订单状态不更新。

### 根因
1. **RSA 公钥在 Node 24/OpenSSL 3.x 下解码失败**：`DECODER routines::unsupported`
2. **MD5 降级也失败**：回调 body 无 `sign` 字段，与 Authorization header 的 RSA base64 签名字符串对比永远对不上
3. 所有回调被拒 → 订单永远是 pending

### 修复（验签策略重构三阶段）
- **第一优先 MD5**：body 有 `sign` + `terminal_sn` → 查 DB 取 terminal_key → MD5 验签
- **第二备选 RSA**：Authorization header → RSA 验签（Card 支付可能用）
- **最后防线 IP 信任**：`true-client-ip` 匹配 `47.96.x.x`（SQB 阿里云杭州服务器）→ 直接放行
- 日志增加 `verifyMethod` 字段追踪验签方式
- 日志增加 `rsaResult` 字段

### 手动处理
- 用户刚付的订单 `claw-basic-1778095427648`（支付宝 ¥0.10）已通过 replay-callback 手动确认
- 返回：订单已处理，会员已升级 ✅
