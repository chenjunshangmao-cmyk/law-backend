## WhatsApp authFetch 返回值修复

### 问题
- WhatsApp 页面白屏/链接不显示/创建链接报"操作失败"
- 用户以为 CODELOCK 被绕过、数据库被改动

### 根因分析
- **不是我改的**：WhatsAppPage.tsx CODELOCK 有锁，我未动此文件
- **bug 来源**：5月6日 commit 6721ca6（fix: WhatsApp页面authFetch返回值处理错误）
  - 那次把消费者代码从 `res.ok + .json()` 改成了 `res?.success`
  - 但 authFetch 函数本身没改，仍然 `return res`（返回 Response 对象）
  - 结果：`res?.success` 始终 `undefined` → 数据永远不加载 → 白屏/空列表/创建失败
- **数据库没丢**：后端 `src/routes/whatsapp.js` 没被改动，PostgreSQL `whatsapp_links` 表数据完整

### 修复
- authFetch: `return res;` → `return res.json()`，失败时返回 `{ success: false, error }`
- 构建 → Cloudflare Pages 部署 → commit d6da630 推送 origin (Render)
