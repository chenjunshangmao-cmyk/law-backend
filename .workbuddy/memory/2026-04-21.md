## GitHub Push Protection 修复

### 问题
GitHub Push Protection 持续拦截含密钥的旧 commit（`6f56062`、`df8a04a`）导致无法推送代码。

### 解决
1. 执行 `git rebase --onto c9d2b8e 6f56062 df8a04a~1` 重写历史（无效）
2. 发现本地 master 分支也包含 `6f56062`，需从干净 detached HEAD 状态修复
3. 最终方案：`git checkout master && git reset --hard 46d8716`
4. `git push origin master --force` 成功，GitHub master 已更新到干净分支

### 教训
- 删除含密钥文件后，需要同步执行 `git commit --amend` 或 `git rebase` 才能从本地分支清除
- Push Protection 扫描的是所有 reachable commits，不只是新推送的
