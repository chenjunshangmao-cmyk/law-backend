## 支付链路核心修复（2026-04-23 19:03）

### 根本原因
`payment.db.js` 和 `shouqianba.db.js` 使用了**不同的终端文件路径**：
- `shouqianba.db.js` 写入：`/app/data/shouqianba-terminal.json` ✅（activate时）
- `payment.db.js` 读取：`/app/src/data/shouqianba-terminal.json` ❌（空文件）
- 结果：payment找不到终端 → testMode=true → 没有真实payUrl

同时，`createWapPayment`（service层）使用了错误的收钱吧端点（`/upay/v2/wap2` POST），而正确工作的是WAP GET网关（`https://m.wosai.cn/qr/gateway`）。

### 修复内容
1. `payment.db.js`：`TERMINAL_FILE` 路径改为 `__dirname + '../../data/shouqianba-terminal.json'`（与shouqianba.db.js完全一致）
2. `payment.db.js`：移除 `createWapPayment` 调用，改为直接生成WAP URL（GET网关，参数带签名）
3. `payment.db.js`：添加 `import crypto from 'crypto'`
4. 移除未使用的 import：`createWapPayment`, `queryOrder`, `handleNotify`

## 会员状态不同步 Bug 修复（2026-04-23 20:49）

### 症状
支付成功后：
- ✅ 后端日志显示 `✅ 用户升级到 basic`
- ✅ 订单状态变为 `paid`
- ✅ 前端弹出"支付成功！会员已激活"
- ❌ 账号页面仍显示"免费版"

### 根本原因
`/api/auth/profile` 用 `findUserById()` 读**本地 JSON 缓存**（`data/users.json`），而不是 PostgreSQL 数据库。`upgradeUserMembership` 更新了数据库，但 profile 接口永远返回缓存里的旧数据。

### 修复
`backend/src/routes/auth.min.js` 的 `/api/auth/profile` 改为优先查 PostgreSQL：
```js
const pgResult = await pool.query(
  `SELECT id, email, name, membership_type, membership_expires_at
   FROM users WHERE email = $1`,
  [decoded.email]
);
return res.json({ success: true, data: { user: {
  plan: dbUser.membership_type || 'free',
  membershipType: dbUser.membership_type || 'free',
  ...
}}});
```
同时 `upgradeUserMembership` SQL 改用 `id::text = $3`（与其他地方一致）。

## YouTube OAuth 修复（2026-04-23 17:54）

### 问题
Google OAuth 报错 `400: redirect_uri_mismatch`

### 原因
后端生成的 `redirect_uri` 包含查询参数 `?mode=popup`，但 Google Cloud Console 里配置的 URI 不包含参数，导致不匹配。

### 修复
- 修改 `auth.youtube.js`：
  - 移除 `redirect_uri` 中的 `mode` 查询参数
  - 将 `mode` 信息通过 `state` 参数传递
  - 回调时从 `state` 中解析 `mode`
