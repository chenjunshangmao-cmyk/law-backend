# 2026-04-19 工作日志

## 会员认证 + 支付系统一次性修复完成

### 问题诊断
- **会员认证 401**：新注册用户无法通过认证（/api/membership/create、/api/payment/create 返回 401）
- **根本原因**：`dataStore.js` 的 `findUserById` 使用 `pool.query` 异步查 PostgreSQL，失败时返回 null；但 `auth.min.js` 注册用户写入的是 JSON 文件（`data/users.json`），两边数据不同步

### 修复方案（最小化稳健）
1. `src/services/dataStore.js`：恢复同步 JSON 文件读取，移除 pool 查询依赖
   - `auth.min.js` 写 JSON（`data/users.json`），`dataStore.js` 读同一文件
   - 两边共用同一数据源，完全同步，不再有异步问题
2. `src/middleware/auth.js`：保留 try-catch，防止意外错误导致服务崩溃

### 修复结果
- ✅ 注册: 201（用户创建成功）
- ✅ /api/membership/create: 200（订单创建成功）
- ✅ /api/payment/create: 200（支付链接生成成功，测试模式）
- 订单号格式: `CLAW{timestamp}{随机数}`
- 金额: 199.00元（基础版）
- 收钱吧未配置时自动降级为测试模式，返回模拟支付链接

### Git 提交
- Commit: `ad77fad` - fix(auth): dataStore.js改回同步JSON查询

### 教训
- pool 异步查询失败时静默返回 null，导致认证失败
- 不要在认证关键路径上引入可能失败的外部依赖
- 同步 JSON 读写比数据库更可靠（对于 auth.min.js 这种单写多读场景）
