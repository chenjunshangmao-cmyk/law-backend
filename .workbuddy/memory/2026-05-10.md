### AI直播推流代理修复 (v2026.05.10.009)
- **问题诊断**：用户反馈点开播后推流秒断
- **Bug #1（致命）**：ownProxyUrl 未被解析为 proxyConfig
  - LiveStreamEngine.prepare() 只认 proxyConfig（代理池），不认 ownProxyUrl（用户自带）
  - 修复：prepare() 新增 ownProxyUrl 解析 → proxyConfig（URL解析 socks5://...）
  - live-stream.js 同步解析 ownProxyUrl 到 engine.proxyConfig
- **Bug #2（架构问题）**：SOCKS5 代理无法用于 FFmpeg RTMP 推流
  - FFmpeg librtmp 不支持 SOCKS5，只认 HTTP CONNECT 代理（RTMP_PROXY env）
  - 修复：RTMPPusher.js v2.0 新增 socat TCP 隧道方案
    - socat TCP-LISTEN:localPort → SOCKS5 → RTMP server
    - FFmpeg 推流到 127.0.0.1:localPort，socat 透明转发
    - socat 不可用时：输出清晰指引（推荐 TUN 模式 / 安装 socat / Proxifier）
  - 停止时自动清理 socat 进程
- **推流方式推荐**：国内 Windows → 开 Clash/V2Ray TUN 模式（系统级代理）→ 自带代理填 SOCKS5 地址 → 开播

### AI直播报错修复 (v2026.05.10.017) 🔧
- **报错**：`/api/live-stream/generate-script` 返回 "Cannot convert argument to a ByteString..."
- **根因 #1**：`src/index.db.js` 强制设 `HTTP_PROXY=http://127.0.0.1:6789`，Render 上无 Clash 代理导致所有 fetch 调用报 ByteString 编码错误（Node.js undici 尝试走代理但代理不存在）
- **根因 #2**：`src/routes/live-stream.js` preview-voice 端点调用 `textToSpeech` 但缺少 import → `textToSpeech is not defined`
- **根因 #3**：`src/services/writer/NovelGenerator.js` 直接调 DeepSeek API，受代理影响全挂（连带 /api/writer/* 全部报错）
- **修复 #1**：index.db.js 加 `IS_RENDER` 检测，Render 环境 delete HTTP_PROXY/HTTPS_PROXY，本地才走 Clash
- **修复 #2**：live-stream.js 补充 `import { textToSpeech } from '../services/avatar/TTSEngine.js'`
- **修复 #3**：NovelGenerator.js 改用 AIGateway 网关（`callLLM` → `gateway.chat()`），自动 fallback 多 provider
- **验证**：3个端点全部恢复 — generate-script / preview-voice / writer/outline 均返回 success
