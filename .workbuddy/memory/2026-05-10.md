### AI直播推流代理修复 (v2026.05.10.009)
- **问题诊断**：用户反馈点开播后推流秒断
- **Bug #1（致命）**：ownProxyUrl 未被解析为 proxyConfig
  - LiveStreamEngine.prepare() 只认 proxyConfig（代理池），不认 ownProxyUrl（用户自带）
  - 修复：prepare() 新增 ownProxyUrl 解析 → proxyConfig（URL解析 socks5://...）
  - live-stream.js 同步解析 ownProxyUrl 到 engine.proxyConfig
- **Bug #2（架构问题）**：SOCKS5 代理无法用于 FFmpeg RTMP 推流
  - FFmpeg librtmp 不支持 SOCKS5，只认 HTTP CONNECT 代理（RTMP_PROXY env）
  - 修复：RTMPPusher.js v2.0 新增 socat TCP 隧道方案
    - socat TCP-LISTEN:localPort → SOCKS5 → RTMP server
    - FFmpeg 推流到 127.0.0.1:localPort，socat 透明转发
    - socat 不可用时：输出清晰指引（推荐 TUN 模式 / 安装 socat / Proxifier）
  - 停止时自动清理 socat 进程
- **推流方式推荐**：国内 Windows → 开 Clash/V2Ray TUN 模式（系统级代理）→ 自带代理填 SOCKS5 地址 → 开播
