# 支付系统修复 - 运营部署说明

## 需要更新的文件（2个）

| 文件路径 | 说明 |
|---------|------|
| `backend/src/services/shouqianba.js` | 收钱吧支付核心，修复了WAP支付URL生成 |
| `backend/src/routes/payment.db.js` | 支付路由，修复了3个bug |

## 具体修复内容

### 1. shouqianba.js
**修复内容**：原 `createWapPayment` 函数从不做真实API调用，`sn` 永远为 null，导致回调永远无法触发。

**修复后**：
- 正确构造 WAP 支付 URL（GET 方式）：`https://m.wosai.cn/qr/gateway?client_sn=xxx&sign=xxx`
- MD5 签名算法正确（ASCII排序 + terminal_key）
- 前端可直接跳转或生成二维码

### 2. payment.db.js
**修复内容**：
- **Webhook 路径**：`/webhook/shouqianba` → `/shouqianba`（避免路径重复）
- **会员升级字段**：`plan/expires_at` → `membership_type/membership_expires_at`（匹配数据库）
- **额度表名**：`user_quotas` → `quotas`（匹配 init-db.js）

## 部署步骤

1. 用 `backend/deploy-files/shouqianba.js` 替换 `backend/src/services/shouqianba.js`
2. 用 `backend/deploy-files/payment.db.js` 替换 `backend/src/routes/payment.db.js`
3. 推送到 GitHub 触发 Render 自动部署
4. 或在 Render Dashboard 点击 **Manual Deploy**

## 验证方式

部署后访问：
```
GET https://claw-backend-2026.onrender.com/api/health
```

检查日志中是否有 `【收钱吧WAP】` 开头的日志输出。

## 测试支付

1. 登录 Claw 网站
2. 进入「会员中心」选择套餐
3. 点击购买，查看是否生成收钱吧支付二维码
4. 完成支付后，检查会员是否自动升级
