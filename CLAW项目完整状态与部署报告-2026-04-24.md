## 六、核心问题诊断

### 问题1：complete-deploy 不完整
- **现象**：`complete-deploy/` 只有31个文件，用户要求的是76个文件
- **原因**：Cloudflare Pages 在构建时生成额外的图标 chunk 文件（Vite 代码分割），这些文件不提交到 Git
- **不影响**：前端构建正常，部署到 Cloudflare 后文件完整

### 问题2：无法下载 e5549ccf 版本的完整构建产物
- **原因**：`wrangler pages deployment download` 命令只能下载 git 追踪的文件，无法下载构建产物
- **绕过方案**：重新从源码构建并部署

### 问题3：Git 分支分叉
- **现象**：`Your branch and 'origin/master' have diverged`
- **原因**：本地和远程各自有独立提交
- **风险**：直接 push 会冲突

## 七、前端部署方法

### 方法A：通过 GitHub 自动构建（推荐）

**原理**：推送到 `claw-github` → Cloudflare Pages 自动触发构建 → 自动部署

```bash
# 1. 确认 SSH key 存在
cat ~/.ssh/id_ed25519.pub

# 2. 推送到 claw-github
cd C:\Users\Administrator\WorkBuddy\Claw
git checkout master
git add frontend/src/
git commit -m "chore: sync frontend source"
git push claw-github master
```

**Cloudflare 会自动：**
1. 检测到 claw-github 有新提交
2. 运行 `npm run build`（Vite 构建）
3. 生成完整的 chunk 文件（包含所有图标）
4. 部署到 `https://claw-app-2026.pages.dev`

**预计时间**：1-3 分钟

---

### 方法B：本地构建 + wrangler 直接部署

```bash
# 1. 本地构建前端
cd C:\Users\Administrator\WorkBuddy\Claw\frontend
npm install
npm run build

# 2. 复制构建产物到 complete-deploy
# Vite 输出到 frontend/dist/，需复制到项目根目录 complete-deploy/

# 3. 用 wrangler 直接部署
npx wrangler pages deploy complete-deploy --project-name=claw-app-2026
```
