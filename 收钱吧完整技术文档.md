## 五、签名算法

### WAP 支付（MD5）
```
规则：排除 sign/sign_type，按 key 字典序拼接，末尾加 &key=terminalKey，MD5 大写

步骤：
1. 排除 sign 和 sign_type 字段
2. 剩余参数按 key 字典序排序
3. 拼接成 key1=val1&key2=val2&...
4. 末尾加上 &key=terminalKey
5. MD5(内容).toUpperCase()

示例参数：
{ terminal_sn: '100111220054389553', client_sn: 'CLAW123', total_amount: '19900', subject: 'Claw 基础版', return_url: '...', notify_url: '...', operator: 'claw_admin' }
```

### RSA（Card 支付，暂未启用）
```
格式：MD5(参数拼接 + &ckey= 结尾)
```

## 六、三层状态判断（验收标准）

回调处理必须严格按三层判断：

| 层级 | 字段 | 判断条件 | 处理 |
|------|------|---------|------|
| 第一层 | `result_code` | `=== 'SUCCESS'` | 通讯层：收到响应 |
| 第二层 | `biz_response.result_code` | `=== 'SUCCESS'` | 业务层：业务处理成功 |
| 第三层 | `order_status` | `=== 'PAID'` | 订单层：资金已到账 |

```javascript
// 第一层：通讯层
if (result.result_code !== 'SUCCESS') return res.send('fail');

// 第二层：业务结果层
const biz = result.biz_response || result;
if (biz.result_code !== 'SUCCESS') return res.send('fail');

// 第三层：订单状态层
if (biz.order_status === 'PAID') {
  // 更新数据库：status='paid', 激活会员
} else if (biz.order_status === 'CLOSED') {
  // 标记订单关闭
}
```

## 七、完整支付流程

```
用户点击购买
    ↓
前端 POST /api/payment/create { plan: 'basic' }
    ↓
后端：
  1. 读取终端 terminal.json（terminal_sn + terminalKey）
  2. 生成订单号 CLAW{timestamp}{3位随机}
  3. 构造 WAP 参数（total_amount = 19900，单位：分）
  4. MD5 签名（terminalKey）
  5. 生成支付 URL：https://m.wosai.cn/qr/gateway?...
  6. 保存订单到 payment_orders 表
    ↓
返回 payUrl，前端生成二维码
    ↓
用户微信/支付宝扫码
    ↓
收钱吧支付成功
    ↓
收钱吧 POST /api/webhook/shouqianba（回调）
    ↓
后端回调处理（payment.db.js）：
  1. 从 callback params 读 terminal_sn
  2. 查数据库 shouqianba_terminals，按 terminal_sn 找 terminalKey
  3. MD5 验签（用查到的 terminalKey）
  4. 三层状态判断（result_code → biz_response.rc → order_status）
  5. 验签通过：
     - 订单 status → 'paid'
     - 用户 membership_type → 对应套餐
     - 记录 paid_at
     - 返回 'success' 给收钱吧
    ↓
前端轮询检测到 status=paid
    ↓
弹窗提示"支付成功！会员已激活"
```

## 九、数据库表

### shouqianba_terminals（终端表）
```sql
CREATE TABLE shouqianba_terminals (
  id SERIAL PRIMARY KEY,
  device_id VARCHAR(50) UNIQUE,
  terminal_sn VARCHAR(50) UNIQUE,
  terminal_key VARCHAR(100),
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW()
);
```

### payment_orders（订单表）
```sql
CREATE TABLE payment_orders (
  id SERIAL PRIMARY KEY,
  order_no VARCHAR(50) UNIQUE,
  shouqianba_sn VARCHAR(50),
  user_id VARCHAR(100),
  plan_type VARCHAR(30),
  plan_name VARCHAR(50),
  amount INTEGER,          -- 单位：分
  status VARCHAR(20),      -- pending/paid/closed/refunded
  paid_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```
